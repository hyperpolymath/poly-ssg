= poly-ssg Design Specifications Index
:toc:

== Overview

This directory contains design specifications for 27 SSG repositories in the hyperpolymath ecosystem. Each spec recommends one of four verdicts:

[cols="1,3,1"]
|===
|Verdict |Meaning |Count

|*ENGINE*
|Standalone SSG implementation in a distinct language/paradigm
|14

|*THEME*
|Visual theme for an existing engine
|2

|*INTEGRATION*
|Integrates with another hyperpolymath project
|6

|*ARCHIVE*
|Not worth pursuing; archive the repository
|5
|===

== Recommended Prioritization

=== Phase 1: Foundation (Months 1-3)
1. **eclipse-ssg** (Pony) - Unique actor model
2. **chicxulub-ssg** (Bash) - Zero dependencies, practical

=== Phase 2: Practical Engines (Months 4-6)
3. **tyrano-ssg** (REXX) - Well-developed README
4. **divisionone-ssg** (COBOL) - Genuine niche
5. **anvil-ssg** (Ada/SPARK) - Safety-critical niche

=== Phase 3: Array Languages (Months 7-9)
6. **jura-ssg** (J) - ASCII-based array
7. **iota-ssg** (APL) - Unicode array

=== Phase 4: Remaining Engines (Months 10-15)
8. **canon-ssg** (Standard ML)
9. **pharos-ssg** (Pharo)
10. **redshift-ssg** (Red)
11. **warp-ssg** (Factor)
12. **vindaloo-ssg** (Curry)

=== Ongoing: Integrations and Themes
- Extract plugins from dei-ssg, obli-ssg, ultimatum-ssg
- Merge noteg-ssg into anvil-ssg
- Develop themes for reliquary-ssg, tripos-ssg
